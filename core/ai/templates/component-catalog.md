# Component Catalog 模板

## 目的

提供项目级 Component Catalog 的通用数据结构模板，用于沉淀可检索、可维护、可验证的组件资产索引。

本模板只定义通用结构，不保存具体项目的真实组件清单。

## 使用场景

- 项目首次建立组件资产目录
- 组件新增、接口变更、迁移、删除后更新 Catalog
- Component Analyst 需要输出可落地的组件资产条目
- Reviewer 需要检查组件资产文档与源码是否一致

## 输入要求

| 输入 | 是否必须 | 说明 |
|------|----------|------|
| 组件名称 | 必须 | 组件文件名或导出的组件名称 |
| 分类 | 必须 | 通用组件 / 业务组件 / 页面私有组件 / 第三方封装组件 |
| 源码路径 | 必须 | 相对项目根目录的源码路径 |
| 职责 | 必须 | 组件解决的问题 |
| 接口信息 | 必须 | Props / Emits / Slots / Expose |
| 关系信息 | 必须 | 依赖、引用、相似或替代组件 |
| 验证信息 | 必须 | lastVerified、验证来源、维护状态 |

## 工作流程

1. 读取 `core/docs/directory.md` 判断组件分类
2. 使用 GitNexus 校验组件路径、引用关系、影响范围和相似组件
3. GitNexus 不可用时，使用 `rg`、imports 和文件读取降级校验
4. 按本模板填写组件条目
5. 不确定字段标记为“待验证”
6. 更新项目级 Catalog 索引、快照和变更记录

## MCP要求

| MCP | 使用时机 |
|-----|----------|
| GitNexus | 校验组件路径、引用关系、影响范围、相似组件 |
| Context7 | 第三方 UI 组件库或框架 API 不确定时 |
| Playwright | 仅在实际页面实现后验证渲染和交互 |

不得修改或实现 MCP 配置。

## 输出格式

```text
## 组件名称
[ComponentName]

## 分类
[通用组件 / 业务组件 / 页面私有组件 / 第三方封装组件]

## 源码路径
[path/to/component.vue]

## 职责
[组件职责]

## Props
[已验证字段或待验证]

## Emits
[已验证字段或待验证]

## Slots
[已验证字段或待验证]

## Expose
[已验证字段或待验证]

## 依赖关系
[依赖组件、composables、utils、第三方库]

## 引用关系
[引用方、页面、模块、影响范围]

## 相似或替代组件
[可替代组件、相似组件、差异点]

## 复用决策
[复用 / 扩展 / 包装 / 新建 / 拆分]

## 设计组件映射
[设计组件或待验证]

## 状态
[active / deprecated / migrated / removed]

## lastVerified
[YYYY-MM-DD]

## 验证来源
[GitNexus / rg / 文件读取 / 待验证]

## 变更记录
[日期、变更说明]
```

## 注意事项

- Catalog 是源码索引与摘要，不替代源码事实
- 不确定字段必须写“待验证”
- 不把实际项目组件清单写入 core
- 公共组件变更必须记录影响范围
- 页面私有组件跨模块复用前必须评估是否提升层级
