# design-analysis

## 目的

分析设计稿或 UI 输入，提取页面结构、布局层级、组件拆分、样式规范、Design Token 建议、组件映射建议，并输出 Developer 可执行任务。

本 Skill 是工具无关定义，不绑定具体设计工具。

## 使用场景

- 用户提供 Figma 链接
- 用户提供图片、页面截图或 UI 说明
- 用户要求页面还原、视觉还原、截图还原
- 需要从设计稿生成页面或组件开发任务
- 需要判断设计稿与现有组件、模板的匹配关系

## 输入要求

| 输入 | 是否必须 | 说明 |
|------|----------|------|
| Figma 链接 | 可选 | 设计稿链接或节点链接 |
| 图片 | 可选 | 静态设计图或视觉参考 |
| 页面截图 | 可选 | 需要还原的页面截图 |
| UI 说明 | 可选 | 用户提供的布局、交互或样式说明 |
| 目标页面 | 必须 | 要实现或还原的页面、模块、路由或组件 |
| 项目约束 | 可选 | UI 库、预设、响应式、主题、权限等要求 |

至少需要提供一种设计输入：Figma 链接、图片、页面截图或 UI 说明。

## 执行流程

1. 获取设计信息
   - 使用可用的设计输入 MCP；例如 Figma MCP
   - MCP 不可用时，基于图片、截图或 UI 说明降级分析
2. 分析页面结构
   - 识别页面区域、导航、内容区、操作区、弹层或状态区
3. 分析布局层级
   - 提取栅格、对齐、间距、响应式断点和滚动区域
4. 识别组件
   - 区分通用组件、业务组件、页面私有组件和第三方 UI 组件
5. 匹配已有组件
   - 优先使用 GitNexus 查找项目已有页面、组件和样式实现
6. 输出样式规范
   - 提取颜色、字体、字号、行高、间距、圆角、边框、阴影等
7. 输出 Design Token 建议
   - 仅提出建议，不直接创建 token 文件
8. 输出组件映射建议
   - 映射到已有组件、需要新增组件或模板
9. 输出 Developer 可执行任务
   - 明确 Skill、Template、文件范围和验证建议

## MCP 要求

遵循 `core/AGENTS.md` Section 6 的全局 MCP 优先级与降级策略（设计任务前置设计输入 MCP，例如 Figma MCP，用于获取设计结构、尺寸、组件和样式信息）。

## 输出格式

```text
## 页面结构
[页面区域、信息架构、状态和交互入口]

## 布局层级
[栅格、对齐、间距、响应式、滚动和层级关系]

## 组件拆分
[通用组件、业务组件、页面私有组件、第三方组件]

## 样式规范
[颜色、字体、间距、圆角、边框、阴影、状态样式]

## Design Token 建议
[token 名称建议、语义、来源、适用范围]

## 组件映射建议
[已有组件匹配、新增组件建议、Template 建议]

## Developer 可执行任务
[文件范围、Skill、Template、实现步骤、验证建议]
```

## 注意事项

- 不绑定具体设计工具
- 不在分析阶段修改代码
- 不凭空补全不可见设计细节
- 设计信息缺失时必须列为风险
- 优先复用项目已有组件和模板
- Design Token 只输出建议，不直接创建或修改 token 文件
