# bug-fix

## 目的

定义前端 Bug 定位与修复流程，作为 Claude 与 Codex 共享的工具无关 Skill。

## 使用场景

- 用户报告功能异常
- 页面渲染异常或数据显示错误
- 控制台出现错误
- 异步、响应式、类型或交互问题需要定位

## 输入要求

| 参数 | 是否必须 | 说明 |
|------|----------|------|
| Bug 现象描述 | 必须 | 出现了什么问题 |
| 复现步骤 | 必须 | 如何触发该问题 |
| 报错信息 | 可选 | 控制台错误栈或日志 |
| 预期行为 | 可选 | 正常情况下应该是什么 |
| 影响范围 | 可选 | 哪些页面或功能受影响 |

## 执行流程

1. 明确实际行为和预期行为差异
2. 根据现象、路由、报错栈或功能描述定位相关代码
3. 分析调用链和影响范围
4. 阅读相关文件，确认根因，不猜测
5. 制定最小修复方案，不引入无关重构
6. 实施修复并执行类型检查
7. 按原复现步骤验证，并检查相关回归
8. 按输出格式总结

## MCP 要求

遵循 `core/AGENTS.md` Section 6 的全局 MCP 优先级与降级策略。GitNexus 用于定位相关函数、调用链、引用关系和影响范围。

## 输出格式

遵循 `core/AGENTS.md` Section 8 全局响应格式（理解 / 计划 / 变更 / 验证 / 遗留风险）。「理解」聚焦 Bug 现象和根因，「验证」说明复现步骤重测结果。

## 注意事项

- 不在 Bug 修复中顺手重构
- 不扩大修改范围
- 不基于猜测修复
- 异步、响应式和空值边界必须重点检查
- 不修改 Claude 专属 `.claude/skills/*/SKILL.md`
