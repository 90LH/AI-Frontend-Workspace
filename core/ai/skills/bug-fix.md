# bug-fix

## 目的

定义前端 Bug 定位与修复流程，作为 Claude 与 Codex 共享的工具无关 Skill。

## 使用场景

- 用户报告功能异常
- 页面渲染异常或数据显示错误
- 控制台出现错误
- 异步、响应式、类型或交互问题需要定位

## 输入要求

| 参数 | 是否必须 | 说明 |
|------|----------|------|
| Bug 现象描述 | 必须 | 出现了什么问题 |
| 复现步骤 | 必须 | 如何触发该问题 |
| 报错信息 | 可选 | 控制台错误栈或日志 |
| 预期行为 | 可选 | 正常情况下应该是什么 |
| 影响范围 | 可选 | 哪些页面或功能受影响 |

## 执行流程

1. 明确实际行为和预期行为差异
2. 根据现象、路由、报错栈或功能描述定位相关代码
3. 分析调用链和影响范围
4. 阅读相关文件，确认根因，不猜测
5. 制定最小修复方案，不引入无关重构
6. 实施修复并执行类型检查
7. 按原复现步骤验证，并检查相关回归
8. 按输出格式总结

## MCP 要求

| MCP | 使用时机 |
|-----|----------|
| GitNexus | 定位相关函数、调用链、引用关系和影响范围 |
| Context7 | 判断是否为框架或第三方 API 误用时查询官方文档 |
| Playwright | 复现 Bug、验证修复、检查主要回归路径 |

调用顺序：GitNexus -> Context7 -> Playwright。

只有 MCP 不可用或目标仓库未索引时，才允许降级到 `rg`、读取 imports、类型检查和手动验证步骤。

## 输出格式

```text
## 理解
[Bug 现象、复现方式和根因]

## 计划
[修复方案和影响范围]

## 变更
[已修改的文件路径和内容摘要]

## 验证
[复现步骤重测、回归检查或降级验证说明]

## 遗留风险
[同类问题、未覆盖边界或验证限制]
```

## 注意事项

- 不在 Bug 修复中顺手重构
- 不扩大修改范围
- 不基于猜测修复
- 异步、响应式和空值边界必须重点检查
- 不修改 Claude 专属 `.claude/skills/*/SKILL.md`
