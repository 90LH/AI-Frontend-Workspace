# vue-page-create

## 使用场景

新建一个 Vue3 路由页面时使用。

触发条件：
- 用户要求新建页面
- 需要创建对应路由的视图文件
- 需要搭建页面骨架（布局、状态、接口调用）

---

## 输入要求

调用前需提供：

| 参数 | 是否必须 | 说明 |
|------|----------|------|
| 页面名称 | 必须 | 如 `UserList`、`OrderDetail` |
| 路由路径 | 必须 | 如 `/user/list` |
| 页面功能描述 | 必须 | 用于理解业务逻辑 |
| 依赖接口 | 可选 | 已有 API 路径或说明 |
| 关联组件 | 可选 | 需要引入的已有组件 |

---

## 执行步骤

1. **分析需求**
   - 理解页面功能和数据流
   - 确认页面是列表、详情、表单还是混合型

2. **检查已有资源**
   - 搜索现有同类页面作为参考
   - 检查可复用的组件和 hooks
   - 检查已有 API 定义

3. **确定文件位置**
   - 遵循项目 `src/views/` 目录规范
   - 确认路由注册位置

4. **创建页面文件**
   - 使用 `<script setup lang="ts">` 语法
   - 使用 Composition API
   - 引入必要的 Pinia store
   - 引入必要的 Vue Router composables

5. **实现页面逻辑**
   - 数据获取（onMounted 或 watchEffect）
   - 状态管理
   - 事件处理

6. **验证**
   - 确认 TypeScript 无报错
   - 确认路由可正常访问

---

## MCP 调用规则

| MCP | 使用时机 |
|-----|----------|
| **GitNexus** | 查找项目中已有同类页面的代码结构 |
| **Context7** | 查询 Vue3 / Pinia / Vue Router 官方文档用法 |
| **Playwright** | 页面创建后在浏览器中验证渲染结果 |

调用顺序：GitNexus（了解已有结构）→ Context7（确认 API 用法）→ Playwright（验证结果）
