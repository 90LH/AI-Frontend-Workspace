# Component Catalog 变更记录

## 目的

记录项目级 Component Catalog 的创建、维护、快照、校验和组件资产变更历史。

## 使用场景

- 追踪组件资产目录维护历史
- 记录组件新增、接口变更、迁移、删除、引用变化
- Reviewer 校验 Catalog 与源码一致性
- Component Analyst 进行组件资产差异分析

## 输入要求

| 输入 | 是否必须 | 说明 |
|------|----------|------|
| 变更日期 | 必须 | Catalog 维护日期 |
| 变更范围 | 必须 | 索引、组件条目、快照或验证方式 |
| 验证来源 | 必须 | GitNexus、rg、文件读取、Reviewer |

## 工作流程

1. 每次 Catalog 维护后新增中文记录
2. 只记录实际完成的变更
3. 标明验证方式和范围
4. 不改写历史记录

## MCP要求

| MCP | 使用时机 |
|-----|----------|
| GitNexus | 校验组件关系和影响范围 |
| Context7 | 本记录通常不需要 |
| Playwright | 仅在页面实现后记录验证结果 |

## 输出格式

```text
## YYYY-MM-DD

### 新增

- 新增：...

### 验证

- 验证：...
```
